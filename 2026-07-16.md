## Tablet RGB LEDs wired to the engine
The RK3576S panel's [[Tablet Wall Panel|status LED strip]] now reflects engine state. `/usr/local/bin/led-panel.py` — a **root Python daemon** (stdlib only) driven by `led-panel.service` (systemd, enabled → boot-persistent). Running as root meant the planned NOPASSWD sudoers wasn't needed.

- **Mapping (priority high→low):** EVAC → red, paging live → green, bell ringing → blue, engine unreachable → amber (fault). **Idle = dark** (Jamie removed the idle glow). Paging deliberately outranks ringing (a live mic is the salient state, and the bell is held by the page anyway).
- **All active states breathe** — a cosine pulse with an eased low-dwell and a 4% floor. **EVAC breathes faster (~1.6s)** to read as urgent; the others are ~4s (`BREATH_PERIOD_STATE` override).
- For a smooth 30fps it **writes the PWM `duty_cycle` sysfs directly** (init once, then modulate) instead of shelling out to `setcolor` each frame; the engine poll runs in a background thread so a slow request can't stutter the breath. Engine URL tries `.42:8095` then `.41:8095` (the piface Pi is DHCP and moves).
- **Verified on real hardware by reading PWM duty back** (can't see the LED remotely): idle = all off; EVAC = red channel sweeping 960784↔3921 over ~1.6s while G/B stay off = a clean breathing red. Source backed up to `Tablet\led\`.

## Tablet kiosk: killed the long-press menu + hid the cursor
Long-press on the panel popped Chromium's touch **context menu**, and a mouse cursor showed. Fixed in the panel's own code (holds across every screen): `preventDefault` on `contextmenu`/`dragstart`/`selectstart`, plus `cursor:none !important` and `-webkit-touch-callout:none`. Verified live with a synthetic right-click — no menu window appeared (before, it would). `unclutter` was already hiding the OS pointer; the CSS reinforces it in-page.

## Console UI clean-ups (all three consoles)
A batch of layout fixes the installers wanted, each committed + rolled out to all three:
- **Calendar page:** moved the month/year grid **above** the selected-day editor (you now see the calendar first, tap a day, edit below) — fixed the awkward "tap a day to edit it above" flow.
- **Nav:** pushed **Log and Help to the far right** (a spacer splits them from the primary tabs).
- **Setup & Help:** replaced the stacked second row of buttons with a **left sidebar**. First attempt put it inside the centered content (pushed the cards right); Jamie wanted the **empty left gutter** used instead. So on wide screens (≥1720px, where a gutter exists) the sidebar is `position:fixed` in the gutter and the content keeps its full centered width; narrower screens fall back to an in-flow column. **Pinned** so it stays visible while scrolling (below the sticky header, calc'd left). Showed a mockup + rendered images first for sign-off — and learned the CLI user only sees an image if I save the PNG to the Desktop and open it (tool-viewed images aren't visible to them).
- **Secondary NTP server** (Setup → School): timesyncd's `NTP=` takes a space-separated list and fails over in order, so a secondary is `NTP=<primary> <secondary>` (not `FallbackNTP=`, which is ignored when `NTP=` is set). Backend (`system.py`/`service.py`/`webapp.py`) + a Primary/Secondary input pair. **Verified live** on the office Pi (wrote both servers, parsed both back, guard rejects a secondary with no primary), then **restored the Pi's original single-server config** so nothing on the appliance changed.
- **Calendar month view** was capped at `max-width:840px` while the year grid fills the width — dropped the cap so both views are the same size.
