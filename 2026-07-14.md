# 2026-07-14 — Bell Commander: tablet wall panel bring-up

Started the **tablet UI** track for [[Beyond Bell Commander]]. A Chinese wall-mounted "Smart Home Control Panel" arrived (from supplier Teren Fu) to become the touch front-end. Spent today on hardware bring-up before any UI work. Everything reachable at `192.168.1.40`, SSH key installed.

## The hardware
- **Rockchip RK3576S** (8-core), 10.1" panel, native 800×1280 DSI rotated to **1280×800** landscape (16:10) — that's the tablet UI target resolution.
- 4GB RAM, 14GB eMMC, **Debian 12 bookworm** (XFCE + lightdm autologin), Chromium 132. Android 14 firmware also supplied (dual-boot).
- Rear I/O is the interesting part: **relay output, RS-485 + RS-232, RJ45 with PoE, Type-C OTG, IR, 2×2W speaker, 4-mic array, 5MP camera, temp/humidity + light sensors, RGB LED strip.** So it can be a wall console *and* a bell-ringing output device.

## Getting root (factory image was locked)
The image shipped with **every setuid bit stripped** — `sudo`/`su`/`pkexec`/`passwd` all plain, zero setuid binaries, so no privilege escalation at all. Root fs was normal ext4 (not nosuid) — vendor just built it wrong.
- Found `/etc/init.d/rkaiq_3A.sh` **world-writable (777)** and run **as root** at boot by the enabled `rkaiq_3A.service`. Dropped a `chmod u+s /usr/bin/sudo …` into it, rebooted, setuid restored → sudo works (password `linaro`).
- Cleaned up afterwards: restored the original script and tightened it to 755 to close the hole. *(If it ever gets reflashed, expect the same and repeat.)*

## RGB LED strip — manual bring-up
No driver on Debian (vendor `pwm-four-leds`/`/dev/ledjni` is Android-only), so the strip was dark. Drove it by hand:
- **Enable line = gpio151 (gpio4_23), active-HIGH** (device tree claims active-low — it lies).
- Three unclaimed PWMs, inverse polarity (`duty=0` = full on): **pwmchip1=Green, pwmchip2=Blue, pwmchip3=Red**.
- Wrapped it in `/usr/local/bin/setcolor R G B`. Planned use: red = emergency, blue = ringing.

## Sensor check — all present & working
- 🌡️ **Temp/Humidity**: undriven **SHT3x @ i2c-0 0x44**, read directly → 24.8 °C, 61 %RH.
- 🎤 **Mic array** (4× PDM, card1) and 🔊 **Speaker** (ES8388, card0) — both good (heard the test tone).
- 🖐️ **Proximity** (STK3x1x) works; ambient-lux register needs driver config.
- 📷 **Camera** (GC5035 5MP): capture path works, but Debian is **missing the GC5035 iqfile**, so the ISP output is black. Proved it by grabbing RAW Bayer off the CIF and hand-debayering — got a real image. Clean auto-exposed photos need the tuning file (present on Android).
- 🔥 6 SoC thermal zones, 👆 goodix touch. **Relay** is a bare GPIO on the terminal block, not in the device tree — still to be identified.

## Kiosk
- Timezone → **Australia/Sydney**, hostname → **bell-panel**.
- Built a touch **launcher** (`launcher.html`) + `start-kiosk.sh` + XFCE autostart: Chromium boots fullscreen (dedicated profile, screen-blanking/screensaver/lock disabled) into a BNS-branded page that lets you **choose the engine IP on the panel** — numeric keypad, remembers recent servers, 5-second auto-connect countdown. Tested live: launcher renders, keypad responds to touch. Files backed up under `Tablet/kiosk/`.
