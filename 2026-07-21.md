# 2026-07-21

Bell Commander session, mostly hardware/network follow-up from the office Pi's move plus a real audio-routing bug hunt on the Xilica. Written retrospectively (this note was missed at the time — reconstructed from the session's commit history).

## Office Pi network move
The prototype Pi moved off the office VPN subnet (`172.16.200.217`) onto the bench LAN as `192.168.1.211`. Bench testing continued there for the rest of the day.

## Media player: per-device sample rate
`SoundDeviceBackend` used to open every output device at one shared fallback sample rate. Two USB DACs disagreeing on native rate broke this. Rewritten to open each device independently, falling back to *its own* `default_samplerate` on failure, with `_load()` resampling per-call to whichever rate a given device actually opened at. `7180b6b`.

## Panel discovery replaces hardcoded IPs
The tablet paging station, the LED status panel, and the kiosk launcher each had their own hardcoded appliance IP list — all three had drifted independently as devices moved networks. Replaced with real LAN-broadcast discovery: `engine/bellcommander/discovery.py` (UDP responder on the engine side, port 51888) + `Tablet/net/netpanel.py` (`discover_engines()`, `/discover` HTTP endpoint, last-known-good cache) + `Tablet/kiosk/launcher.html` (discovery-first boot flow, falls back to a picker if multiple engines answer) + `Tablet/led/led-panel.py` (same discovery via the netpanel agent). User explicitly chose real broadcast discovery over mDNS, and a general engine feature over a narrow one-off fix. `d093eb5`, `ab598dc`.

## EVAC tone loops until All Clear
Alert/EVAC tones now repeat continuously while a hold is active (`_is_emergency_alert()`: `hold` or `style == "danger"`, and not released/stopped), instead of playing once. All Clear stops the loop. `7d56556`.

## Preamble routing, EVAC redesign, input matching
- Preamble chime wasn't following the page's zones — now explicitly routed alongside the paging source on `page_start`/`page_stop`. `8f433ab`.
- EVAC redesigned per explicit request: instead of unmuting the paging mic itself, EVAC now opens each zone's `Zone{z}AlertGain` and re-mutes the project's own paging-mic ducking gate — the mic crosspoint itself is untouched. `5a04aab`, `9cd8fe6`.
- Input name matching changed from exact-string to whole-word, case-insensitive — a friendlier commissioned label like "Aux/Spare" was silently failing to match "spare". `843ec77`.
- A mojibake bug in an EVAC-related label was found and fixed via panel screenshots (RTA meters, fonts, external input volume controls all separately verified working on the physical panel this session).

## EVAC audibility bug — the big one
Reported: EVAC audio was reaching "Zone 1 Alert Ducker" but never reaching the zone matrix — no sound. Root-caused through live Designer screenshots + a raw `.pjxml` wiring trace, then confirmed against the official SolaroConsole manual: a **Ducker**'s channel 2 ("Priority") is a pure sidechain trigger and is *never* mixed into the block's output — only used to decide how much to attenuate channel 1. The "Zone N Overall Ducker" block had the alert signal wired into that channel, so `Zone{z}AlertGain` opening/closing had no audible effect no matter what the driver did.

Fixed in the Xilica project design itself (not the driver): replaced "Zone N Overall Ducker" with "Zone N Overall Matrix Mixer" — a real 2-input mixer, both inputs genuinely summed to output. Confirmed audible live at zone 1 after the rewire (a "nothing" listening-test result before the fix, "yes evac" after). No driver code changes were needed for the fix itself, since no control names changed — only the docstring was updated to record the root cause. `46a1e2e`, `f9f0eaa`.

Also reversed part of the earlier redesign on explicit instruction: All Clear should *not* floor `Zone{z}AlertGain` back down again — it's deliberately left open once EVAC has engaged (flooring it again had no audio benefit once the Ducker/Mixer fix landed, only an unwanted "gain going down" visible in Designer).

## Panel: Cancel now releases the hold
EVAC panel's Cancel button used to only stop the currently-playing tone. Now it also calls `/api/evac?active=false` — Cancel fully releases the hold, not just silences it. All Clear remains the option if you also want its confirmation tone. `3d1acca`.
