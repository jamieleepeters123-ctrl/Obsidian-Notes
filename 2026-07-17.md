## Wired ethernet — long saga, simple fix (mine to own)
Plugged the panel into the office LAN and its Network card said "not connected". Chased it hard:
- First bug (real): my network agent hardcoded `wlan0`, so on wired it reported "not connected" even when eth0 was up. Fixed — `primary_dev()` now reports whichever interface is actually up (a connected **ethernet wins over wifi**); the launcher labels it "Wired (Ethernet)". Wifi scan/connect still target wlan0.
- Then eth0 genuinely wouldn't link — `NO-CARRIER`, and my `ethtool` forced-speed/autoneg experiments left the PHY stuck advertising "10baseT-only", which **looked exactly like dead hardware**. I diagnosed a faulty NIC and sent Jamie chasing cables + PoE. **That was wrong.**
- **The actual cause:** the interface was administratively **down** (the r8168 powers its PHY off when down → `carrier 0`, indistinguishable from "no cable"). **Val's fix: `sudo ip link set eth0 up`** on a clean state → PHY powered up, autonegotiated with the switch, linked at **100 Mbps full**, NM connected "Wired connection 1", DHCP 172.16.200.162.
- **Why 100, not gigabit:** gigabit can't be manually forced (802.3 needs autoneg for 1000BASE-T); this link settles at 100 — fine for a control panel.
- **Persistence confirmed by reboot:** eth0 auto-comes-up on boot (NM-managed, `Wired connection 1` autoconnect=yes, no admin-down override). Default route prefers **eth0 metric 100 (primary), wlan0 600 (WiFi failover)** — ideal. No boot safeguard needed.
- **Lesson for the next panel:** for a stuck r8168, try `ip link set eth0 up` and *don't* force speed/duplex before ever suspecting hardware.

## Media player — Bell Commander becomes the player (for the Solaro QR1)
Jamie's spec: 4 audio-out channels (**Ch1 Bells, Ch2 Preamble, Ch3 BGM, Ch4 EVAC**), each with a user-set volume, so we can use DSPs **without** internal media players — next up the Xilica Solaro QR1. Clarified: USB audio interface for the outs; preamble = chime before an announcement; BGM = spare music channel that **stops** when anything else plays.

Built end to end (222 tests):
- **`media.py`** — a role-based `MediaPlayer` over a pluggable backend. Real path = PortAudio (sounddevice/soundfile/numpy, lazy-imported, optional extra `[media]`) with a persistent multichannel stream + mixing callback; `NullBackend` when disabled/dep-less, so AHM sites are untouched (feature is off by default).
- **Wiring:** scheduled + manual bells → Bells channel; announcements/EVAC messages → EVAC channel; a page opening fires the preamble chime; bells a page held get their audio on release; STOP silences everything. BGM ducks automatically.
- **Sound library:** upload from the browser (raw-body POST — dodged the python-multipart dependency), name/type/size validated, stored on the appliance (`engine/media/`, gitignored). Sounds gain a **file** field (the `slot` stays for DSP-player sites).
- **Setup → Media Player page:** enable, audio-device picker, the 4 channels with volume sliders + **Test** (a tone at the slider's live level, so levels can be set by ear), upload/manage files, preamble picker, BGM play/stop. Plus a Media file column on the Sounds table.
- The existing Xilica driver's `trigger_bell` always assumed "the appliance delivers the audio; this driver only routes" — this is that missing half.

Deployed to all three consoles (verified live on the office Pi: upload round-trip, AHM stayed online). **Next:** bench session with the USB interface + QR1 — install the `[media]` extra, pick the device, hear real 4-channel audio, and decide preamble-before-mic sequencing.
