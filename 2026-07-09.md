### Beyond Bell Commander — AHM matrix crosspoint cracked (bell→zone routing)

Reverse-engineered the **AHM matrix crosspoint (source→zone "send") command** — the
last piece for bell-group → zone routing ("Option 3"). Done live against the office
AHM (172.16.200.127) over the Sophos VPN, tshark on the Sophos TAP + System Manager
V1.61. Full detail in [[AHM Reverse Engineering]] and [[Beyond Bell Commander]].

**The command**
- Over AHNet (:51321), open the **"Mixer"** service → write `cmd = 0x5000 + row*0x40 + col`,
  payload `01` = route on / `00` = off. A **16×16 matrix** (objects 0x5000–0x53cf).
- Confirmed live 6× (top-left crosspoint = object 0x5000); matches the state-dump
  readback (0x5000 = 0000 off / 0100 on).

**Why it was hard** — System Manager streams the **mouse-cursor position** to the AHM
continuously while the pointer moves, which buried the real 1-byte click write in every
capture. Holding the mouse **still** around the click isolated it. Also: AHNet handles
reassign every reconnect (match services by name), and the VPN was NOT the problem — it
carried 18k-packet captures and full state dumps faithfully. Took ~10 captures + a
state-diff (crosspoint OFF vs ON reconnect dumps) to converge.

**Built + tested (engine, 84 tests, +6)**
- `ahnet.matrix_object()` + `set_matrix_crosspoints()` (mirror of the zone-rename write),
  `read_matrix_crosspoints()` probe, `probe_matrix.py`.
- `AHMDriver.trigger_bell` now routes a bell to its group's zone crosspoints — **gated
  behind `matrix_source_row` (default off, prototype behaviour unchanged)**, delta-tracked,
  fail-soft (a dead AHNet session never stops a bell).
- Write-up: `Desktop\Beyond Bell Commander\AHM\matrix-crosspoint-FINDINGS.md`.

**To activate routing (next, needs to hear/meter the AHM)** — one live experiment:
play a bell, set a crosspoint from our client, confirm which zone responds. That fixes
the `matrix_source_row` value + the source/zone axis, AND confirms the media player
actually routes through this input×zone matrix (the office AHM's 0x5xxx matrix read back
mostly empty, so worth checking it isn't a fixed player→zone path).
